# 2장 - 객체 생성과 파괴

## 아이템1. 생성자 대신 정적 팩터리 메서드를 고려하라
### 정적 팩터리 메서드
  - **[핵심]** 객체 생성 방식을 캡슐화하여 유연하고 효율적인 객체 생성을 가능하게 함
  - **[이점]**
    - 확장성과 유지보수성을 높임
    - 생성자보다 더 많은 제어권과 유연성을 제공
  - **[주의점]**
    - 정적 팩터리 메서드를 찾기 어려울 수 있음
      - **[해결]** 널리 알려진 정적 팩터리 메서드 이름으로 짓기
    - 정적 팩터리 메서드만 제공하면 하위 클래스를 만들 수 없음
      - **[이점]** 상속을 못하도록 의도할 때 사용 가능
